# 实施计划

## 里程碑 1：私有外壳

目标：
- 在还未接入真实收信前，先把安全的应用外壳搭起来。

范围：
- Wrangler 项目初始化
- D1 迁移执行器
- 管理员初始化命令
- 登录页
- 会话 Cookie 认证
- 仪表盘外壳

完成标准：
- Cloudflare Access 已保护整个应用
- 管理员可以登录和退出
- 未授权用户无法访问应用页面

## 里程碑 2：邮箱管理

目标：
- 安全地创建和管理长期邮箱。

范围：
- 子域名池生成
- 邮箱 CRUD
- 备注与保留策略
- 管理员操作审计日志
- 创建邮箱限流

完成标准：
- 管理员可以创建、暂停、归档和删除邮箱
- 系统会自动分配可用子域名
- 每次邮箱操作都会留下审计记录

## 里程碑 3：入站邮件

目标：
- 接收真实邮件并正确持久化。

范围：
- Email Routing Worker 入口
- MIME 解析
- HTML 清洗
- 原始邮件和附件写入 R2
- 邮件元数据写入 D1
- 更新收件箱计数

完成标准：
- 外部真实邮件能出现在正确的邮箱中
- 原始 `.eml`、文本、HTML 和附件都能被保存
- 已暂停的邮箱会被干净地拒收

## 里程碑 4：阅读体验

目标：
- 让应用具备日常可用性。

范围：
- 收件箱页面
- 邮件详情页
- 搜索与筛选
- 复制验证码 / 复制链接
- 下载原始邮件和附件

完成标准：
- 管理员可以快速找到并阅读任意保存的邮件
- HTML 邮件能安全渲染
- 附件下载稳定可靠

## 里程碑 5：安全加固

目标：
- 降低暴力破解和滥用风险。

范围：
- 登录失败跟踪
- IP 临时封禁窗口
- Worker 侧限流
- 审计日志筛选
- 仪表盘安全告警

完成标准：
- 连续失败登录会触发临时封禁
- 高频创建邮箱会被限流
- 可疑行为能在 UI 中被看见
